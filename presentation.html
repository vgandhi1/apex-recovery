The Opportunity

Every robot failure is a lesson
the AI never learns.

  • ✕ Robot policies trained in simulation routinely fail on the factory floor on conditions the simulator never modeled — unusual lighting, worn parts, unexpected objects.
  • ✕ When a robot fails today, a supervisor walks over, hits E-stop, manually repositions, and marks the episode "FAILED." The episode is discarded.
  • ✕ The human correction that follows — the highest-value training signal that exists — is never recorded.
  • ✓ APEX intercepts this moment. It routes the failing robot to a remote teleoperator in seconds, captures the recovery, tags the failure type, and packages the episode for the training pipeline.

Key design insight: If no operator is immediately available, APEX holds the robot in a safe position with the failure scene preserved — rather than E-stopping and discarding the episode. The operator resolves it when capacity allows.
The Platform

Four integrated components,
one end-to-end system.

📡

Confidence Monitor

Watches the robot policy's confidence score 10 times per second. Fires a recovery alert the moment the score drops below threshold for more than half a second — before the robot hard-stops.

🔀

Operator Routing

Selects the best available teleoperator based on connection latency, task specialization, and current workload. Holds the episode in a queue if no operator is immediately available.

🖥️

Operator Cockpit

A browser-based control interface with live telemetry, a predictive overlay that compensates for camera lag, and a one-click failure tagging form. Designed for a teleoperator, not a software engineer.

📦

Training Data Pipeline

Packages the recovery session — video, robot trajectory, force readings, and operator tags — into a structured file the ML training system can ingest directly, without manual processing.

Operator Experience

Everything the operator needs.
Nothing they don't.

  • 1 Live camera feed with a predictive position overlay — the cockpit shows where the object will be by the time the operator's command arrives, not where it was when the frame was captured.
  • 2 Confidence trend chart showing the 30-second run-up to the failure — so the operator understands the context before taking control.
  • 3 One-click handoff block if the connection is too slow to safely recover — the system protects operators from taking control in conditions where they'll make things worse.
  • 4 Structured failure tagging in under 30 seconds — pre-filled by a classifier, confirmed or overridden by the operator. No free-text required.

Strategic Value

Recovery episodes close the gap between
simulation and the real world.

Robot policies are trained in simulation — but simulation can't model everything. When they hit conditions the simulator missed, they fail. The standard fix is targeted real-world data on exactly those conditions. APEX generates it systematically.

LIGHTING
A scene condition the simulator's renderer never produced. Each episode is a labeled real-world lighting counterexample.
OBJ_NOVEL
An object geometry outside the simulator's asset library. The most valuable out-of-distribution training signal available.
COLLAB_CONFLICT
Dynamic human proximity the sim treated as a static environment. Directly teaches multi-agent awareness.
Each tagged recovery episode is not just a corrected mistake — it is a labeled counterexample to a specific simulator assumption. APEX makes collecting these counterexamples operationally tractable at shift scale.
